Roundtrips and test files

These notes apply to file conversion programs like openbabel ( http://www.sourceforge.net/projects/openbabel) and JUMBO http://www.sourceforge.net/projects/cml).

Roundtripping is the process of converting information from format A through various intermediates and back to A. Ideally there should be no information loss or corruption - in practice there usually is. As an example, convert test.mol (MDL) to test.pdb and then convert test.pdb to test.pdb.mol. How different are test.mol and test.pdb.mol?

The flow of information is as follows:

  • read test.mol, possibly skipping some information
  • transform some information algorithmically (e.g. Kekulization)
  • Store information in data structure (that which cannot be stored is lost
  • Read that information from store which is appropriate to test.pdb. That which cannot be represented in test.pdb is lost
  • Algorithmically convert some information
  • Output test.pdb, possibly with unfilled fields
  • ... halfway ...
  • read test.pdb, possibly skipping some information
  • transform some information algorithmically
  • Store information in data structure (that which cannot be stored is lost
  • Read that information from store which is appropriate to test.mol. That which cannot be represented in test.mol is lost
  • Algorithmically convert some information
  • Output test.mol, possibly with unfilled fields

It is highly probable that information will be lost of corrupted in the process. The purpose of these projects is to create a full enough data structure that enough information can be held. Also we must create common open algorithms and ontologies so it is clear how information is transformed and whether corruption occurs.
